Что тестируем

Минимальный “скелет” проекта, который потом можно расширить под любые сайты и вложенности.

Шаблонность

Header и footer подгружаются из отдельных файлов. Меняешь один раз — обновляется везде.

  • templates/header.html
  • templates/footer.html
  • js/layout.js

Разные устройства

Два отдельных CSS: для ПК и для моб. Никаких общих глобальных стилей на элементы.

  • css/desktop.css
  • css/mobile.css
  • Фиксированная шапка

Вывод данных

Данные подставляются JS по принципу шаблона. Позже добавим: разные профили сайтов и вложенность.

  • HTML блоки
  • XML фиды
  • XLSX экспорт

Как это будет работать дальше

1
Одна “база” данных
Храним товары и характеристики (пока в JS/JSON). В будущем можно подключить любую БД.
2
Профили сайтов
Для каждого сайта задаём правила: какие блоки показать и как группировать характеристики.
3
Генератор вывода
Один JS-генератор собирает HTML/XML/XLSX. Если сервисы упадут — статическая версия останется.

Блоки для визуального теста

Стабильная “эко-версия”

Даже если позже подключим API/БД, эта версия может работать как статический сайт: быстрый, дешёвый, с мгновенным откатом через Git.
